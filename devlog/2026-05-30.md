# 开发日志 2026-05-30

## 完成事项
- [x] 修复 HEIC/HEIF 文件无法预览
  - 根因：服务器将 `image/heic`、`image/heif` 归类为 `type: image`，但浏览器不支持在 `<img>` 标签渲染 HEIC
  - 修复：服务器检测 HEIC/HEIF MIME 类型，归类为 `type: file` 而非 `type: image`
  - 兜底：`<img>` 标签添加 `onerror` 处理器，加载失败时显示文件图标占位
- [x] 修复发送多张图片只显示一张（续昨日）
  - 服务器端验证：多文件上传正常创建所有条目（DB 中均有记录）
  - 前端 `normalizeItem()` + `bindCardActions()` 修复已在上次会话完成
- [x] 修复大文件（>50MB）上传失败
  - 根因：JSON+base64 编码使 68.8MB PDF 膨胀到 ~92MB，超过 50MB JSON body 限制
  - 修复：上传从 `JSON+base64` 改为 `multipart/form-data`（multer），大小限制提升到 500MB
  - 改动：http-server.js（multer 中间件）、api.js（FormData 替代 FileReader+base64）
- [x] 修复中文文件名乱码
  - 根因：部分浏览器 Content-Disposition 文件名发送为 raw UTF-8 字节，busboy 将其误当作 Latin-1 解析，产生双重 UTF-8 编码
  - 修复：新增 `fixFileNameEncoding()` 用 Latin-1→UTF-8 反向解码恢复正确文件名
  - 下载响应头改为 RFC 5987 标准 `filename*=UTF-8''` 编码
- [x] 新增二维码扫码连接功能
  - 安装 `qrcode` 依赖
  - 新增 `/api/qrcode` 端点，返回局域网 URL 和 SVG 二维码
  - 前端顶栏添加 📱 按钮，点击弹出二维码弹窗
  - 手机扫码即可直接打开 ClipSync，无需手动输入 IP

## 遇到的问题
1. **HEIC 预览问题**：Apple HEIC 格式在 Windows Chrome/Electron 中无法原生渲染（需 HEVC 扩展）。解决方案是将 HEIC 作为可下载文件处理，保留文件卡片 UI。

2. **多图片问题**：服务器端 API 测试显示多文件上传完全正常（3 张图 → DB 3 条记录），问题已在上次会话的前端 snake_case/camelCase 修复中解决。用户可能需要刷新浏览器缓存。

3. **大文件上传**：JSON+base64 膨胀 33%，68.8MB → 92MB 超过限制。改用 multipart/form-data 直传二进制完美解决。

4. **中文文件名乱码**：双重 UTF-8 编码是 HTTP Content-Disposition 头中 RFC 5987 编码缺失的经典问题。修复方式：Latin-1→UTF-8 反向解码。
